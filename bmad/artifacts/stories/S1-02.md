# S1-02: GitHub OAuth Device Flow

**Size:** L | **Sprint:** 1 | **Deps:** S1-01

## Description
Implement `auth_engine.svelte.ts` with GitHub Device Flow authentication. Token stored in memory with localStorage fallback.

## Acceptance Criteria
- [ ] `auth_engine` exports: `login()`, `logout()`, `getToken()`, `isAuthenticated` (reactive)
- [ ] Device Flow: user gets a code, opens GitHub, enters code, app polls for token
- [ ] Token stored in memory; persisted to localStorage on page unload
- [ ] `logout()` clears token from memory and localStorage
- [ ] Unauthenticated state shows login screen only
- [ ] Error handling: expired code, denied access, network failure
