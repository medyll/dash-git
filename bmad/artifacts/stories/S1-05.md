# S1-05: SWR Query Cache

**Size:** M | **Sprint:** 1 | **Deps:** S1-04

## Description
Implement `query_cache.svelte.ts` with Stale-While-Revalidate pattern using Svelte 5 runes.

## Acceptance Criteria
- [ ] `cached<T>(key, fetcher, ttl?)` returns `{ data: T, isStale: boolean, refresh() }`
- [ ] Returns cached data immediately if available (even if stale)
- [ ] Background revalidation triggers automatically when TTL expires
- [ ] Reactive: components re-render when fresh data arrives
- [ ] Configurable TTL (default 60s for trees, 30s for PRs)
- [ ] Cache eviction: LRU with max 200 entries
